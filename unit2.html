Huffman Coding

Huffman coding ek data compression technique hai jise kisi data ko compress karne ke liye use kiya jata hai. Iska main objective hota hai ki data ka size chhota karna, taaki storage aur transmission mein space aur time bach sake.

Huffman Coding Kaise Kaam Karta Hai?

  1. Character Frequency: Pehle hum dekhte hain ki har character (letter, number, symbol) ka kitna frequency hai.
  2. Build the Tree: Ab hum ek Huffman Tree banate hain. Is tree mein, har character ko ek binary code diya jata hai.
  3. Create Codes: Tree ko read karte hue, left jaane par 0 aur right jaane par 1 assign karte hain.
  4. Encoding: Jab hume data ko encode karna hota hai, hum characters ko unke corresponding binary codes se replace karte hain.
  5. Decoding: Jab hume data ko decode karna hota hai, hum binary codes ko read karte hain aur unhe original characters mein convert karte hain.

Example:

Agar humara input string hai "aabcc", toh Huffman coding se hume mil sakta hai:

  • a: 0
  • b: 10
  • c: 11
Encoded string: 001011

0/1 Knapsack Problem

0/1 Knapsack Problem ek famous optimization problem hai jisme hume kuch items diye jate hain, jinke weights aur values hoti hain. Hume in items ko ek knapsack (bag) mein rakhna hota hai taaki total value maximize ho, lekin knapsack ki capacity limited hoti hai.

Example:

Maan lijiye, hume 3 items diye gaye hain:

  • Item 1: Weight = 1, Value = 1
  • Item 2: Weight = 3, Value = 4
  • Item 3: Weight = 4, Value = 5
Aur humare knapsack ki capacity hai 4.

Solution:

Is problem ka solution hum dynamic programming se nikalte hain.
Sabse pehle hum ek 2D array banate hain jisme rows items ke liye aur columns knapsack ki capacity ke liye honge.

T[i][w] = max(T[i-1][w], T[i-1][w-weight[i]] + value[i])

Final Output:

Aakhir mein, maximum value hoga 6, jo ki item 2 aur item 3 ko lene se milta hai.

Fractional Knapsack Problem

Fractional Knapsack Problem mein hume items ko fraction (hisso) mein lene ki permission hoti hai. Yani agar hume kisi item ka pura weight nahi le sakte, toh hum uska kuch hissa le sakte hain.

Example:

Maan lijiye, hume 3 items diye gaye hain:

  • Item 1: Weight = 10, Value = 60
  • Item 2: Weight = 20, Value = 100
  • Item 3: Weight = 30, Value = 120
Aur humare knapsack ki capacity hai 50.

Solution:

Pehle, items ko unki value-to-weight ratio ke hisaab se sort karte hain:

  • Item 1: 60/10 = 6
  • Item 2: 100/20 = 5
  • Item 3: 120/30 = 4
Toh, hume item 1 aur item 2 pura lena hai, aur item 3 ka kuch hissa lena padega.

Final Output:

Maximum value hoga 240, jo item 1, item 2, aur item 3 ka 16.67 (1/3) hissa lene se milta hai.

Longest Common Subsequence (LCS)

Longest Common Subsequence (LCS) ek algorithm hai jo do sequences mein common elements ka sabse lamba subsequence dhoondta hai. Yeh subsequence zaroori nahi hai ki contiguous (saath-sath) ho, lekin order wahi rehna chahiye.

Example:

Maan lijiye, hume do strings diye gaye hain: "ABCBDAB" aur "BDCAB".

Solution:

Iska LCS "BCAB" hoga.
Hum is problem ko dynamic programming se solve karte hain. Sabse pehle hum ek 2D array banate hain jisme rows pe first string aur columns pe second string hote hain.

If s1[i] == s2[j] then LCS[i][j] = LCS[i-1][j-1] + 1 Else LCS[i][j] = max(LCS[i-1][j], LCS[i][j-1])

Final Output:

Is example ka LCS length 4 hai, jo ki "BCAB" ko represent karta hai.

Huffman Coding

Huffman coding ek lossless data compression technique hai. Iska main concept yeh hai ki hum frequently used characters ko shorter codes assign karte hain, jabki less frequently used characters ko longer codes assign karte hain. Isse overall data size kam hota hai.

Example Character Frequencies:

  • A: 5
  • B: 7
  • C: 10
  • D: 15
  • E: 20

Huffman Tree Construction:

Huffman tree banane ke liye hume ye steps follow karne hote hain:

  1. Sabse pehle, sabhi characters ko unki frequencies ke saath ek priority queue mein rakhna hota hai.
  2. Phir hum do sabse chhoti frequencies wale nodes ko choose karte hain, unhe combine karte hain ek new node mein, jiska frequency dono ka sum hota hai.
  3. Is new node ko queue mein daal dete hain.
  4. Yeh process tab tak repeat karte hain jab tak sirf ek node nahi bach jata, jo ki root node hoti hai.

Step-by-Step Construction:

1. Start with individual frequencies:
A: 5, B: 7, C: 10, D: 15, E: 20

2. Combine A and B (5 + 7 = 12):
New Node (AB): 12, C: 10, D: 15, E: 20

3. Combine C and (AB) (10 + 12 = 22):
New Node (CAB): 22, D: 15, E: 20

4. Combine D and E:
New Node (DE): 35

5. Finally, combine (CAB) and (DE) to form the root node:
New Node (CABDE): 57

Final Huffman Codes:

Is tarah se codes assign kiye jaate hain:

  • A: 00
  • B: 01
  • C: 10
  • D: 110
  • E: 111

Isliye, Huffman coding se hume efficient aur compact representation milta hai, jo data compression mein madadgar hota hai.

Longest Common Subsequence (LCS)

Longest Common Subsequence (LCS) do strings ke beech ka wo subsequence hai jo dono strings mein common hota hai aur uski length maximum hoti hai. Yeh zaroori nahi ki characters consecutive hon, lekin unka order maintain hona chahiye.

Example Strings:

String 1: 'AGGTAB'
String 2: 'GXTXAYB'

Dynamic Programming Approach:

Is problem ko solve karne ke liye hum dynamic programming ka istemal karte hain. Ismein hum ek 2D array banate hain jisme rows String 1 ke characters ko aur columns String 2 ke characters ko represent karte hain.

Steps to Solve:

  1. Ek 2D array `dp` banate hain jiska size (m+1) x (n+1) hai, jahan m aur n dono strings ki lengths hain.
  2. Array ko 0 se initialize karte hain. dp[i][j] ka matlab hai LCS ki length pehle i characters of String 1 aur pehle j characters of String 2 ke liye.
  3. Agar characters match karte hain, toh dp[i][j] = dp[i-1][j-1] + 1 (pichle LCS mein 1 add karte hain).
  4. Agar characters match nahi karte hain, toh dp[i][j] = max(dp[i-1][j], dp[i][j-1]) (ya toh pichla row ya column ka value lete hain).
  5. Finally, dp[m][n] mein LCS ki length hoti hai.

Code Implementation:


def lcs(X, Y):
    m = len(X)
    n = len(Y)
    dp = [[0] * (n + 1) for _ in range(m + 1)]

    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if X[i - 1] == Y[j - 1]:
                dp[i][j] = dp[i - 1][j - 1] + 1
            else:
                dp[i][j] = max(dp[i - 1][j], dp[i][j - 1])

    return dp[m][n]

# Example Usage
X = "AGGTAB"
Y = "GXTXAYB"
print("Length of LCS is", lcs(X, Y))

Final Result:

Is example ke liye, LCS ki length 4 hai aur wo subsequence hai 'GTAB'.

Fractional Knapsack Problem

(a) Fractional Knapsack Problem and Greedy Algorithm

Fractional Knapsack Problem ek aisa problem hai jisme aapko kuch items diye jaate hain, jinmein har item ka weight aur value hota hai. Aapko in items ko knapsack (bag) mein is tarah se pack karna hai ki total value maximum ho, lekin bag ki capacity ko exceed na karein. Is problem mein aap items ke fractions bhi le sakte hain, yani agar aapko poora item nahi rakhna hai toh uska kuch hissa bhi le sakte hain.

Is problem ko solve karne ke liye Greedy Algorithm ka istemal hota hai. Greedy Algorithm har step par wo choice karti hai jo immediate benefit deti hai. Yani, har bar aapko wo item lena hota hai jiska value-to-weight ratio sabse zyada hota hai.

(b) Example with Item Weights and Values

Maan lijiye hume niche diye gaye items hain:

Item Weight (W) Value (V) Value/Weight Ratio
1 2 10 5
2 3 5 1.67
3 5 15 3
4 7 1 0.14
5 7 6 0.86

Humara goal hai maximum total value hasil karna jab bag ki capacity 10 units ho.

Step-by-Step Calculations:

  1. Items ko unke value-to-weight ratio ke hisaab se arrange karte hain (descending order):
  2. Item 1: 2W, 10V (Ratio = 5) - Total Weight = 2, Total Value = 10
  3. Item 3: 5W, 15V (Ratio = 3) - Total Weight = 7, Total Value = 25
  4. Item 2: 3W, 5V (Ratio = 1.67) - Total Weight = 10, Total Value = 30

Ab, hum bag ki total capacity 10 units hai, toh hum items ko is tarah se select karte hain:

  • Item 1 ko pura lete hain (Weight = 2, Value = 10)
  • Item 3 ko pura lete hain (Weight = 5, Value = 15)
  • Item 2 ko pura lete hain (Weight = 3, Value = 5)

Final Selection:

Humare pass total weight 10 units hai aur total value 30 hai. Isliye optimal selection hai:

  • Item 1: Weight = 2, Value = 10
  • Item 3: Weight = 5, Value = 15
  • Item 2: Weight = 3, Value = 5

Total Value: 10 + 15 + 5 = 30

0/1 Knapsack Problem

(a) Problem Statement and Significance

0/1 Knapsack Problem ek classic problem hai jo decision-making aur optimization mein use hoti hai. Is problem mein aapko kuch items diye jaate hain, jinka specific weight aur value hota hai. Aapko ek knapsack (bag) diya jaata hai jismein aapko items ko is tarah se pack karna hai ki total weight knapsack ki capacity se zyada na ho, aur total value maximum ho.

Is problem ki significance yeh hai ki yeh real-life scenarios mein kaam aati hai, jaise resource allocation, budget management, aur logistics.

Is problem ko solve karne mein kuch challenges hain:

  • Aapko decide karna hota hai ki kaunsa item lena hai aur kaunsa nahi.
  • Items ko fractions mein nahi liya ja sakta, yaani aap poora item ya kuch nahi le sakte hain.
  • Problem ki computational complexity high hoti hai, jo isse solve karna mushkil bana deti hai.
