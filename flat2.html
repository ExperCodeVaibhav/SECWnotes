1. Regular Grammars

Regular grammar ek aisa set of rules hota hai jo regular languages ko define karta hai. Yeh do tarike ke hoti hain: **Right Linear Grammar** aur **Left Linear Grammar**.

**Right Linear Grammar (RLG):** Isme non-terminal symbol hammesha right side par hota hai, jaise:
Example: A → aB | b

**Left Linear Grammar (LLG):** Isme non-terminal symbol hammesha left side par hota hai, jaise:
Example: A → Ba | a

Dono grammars regular languages ko define karte hain, lekin RLG zyada common hoti hai.

2. Regular Expressions

Regular Expressions (RE) ek aisi notation hai jo regular languages ko represent karti hai. Yeh ek tarah se string patterns ko describe karti hai.

Example: Σ = {a, b} ke liye ek regular expression jo 'a' se shuru hoti ho aur koi bhi sequence ho sakta hai:
**Regular Expression:** a(a|b)*

Is expression ka matlab hai ki string 'a' se shuru hogi aur uske baad koi bhi combination ho sakta hai 'a' aur 'b' ka.

3. Equivalence Between Regular Languages

Do regular languages tab equivalent kehlati hain jab dono same set of strings ko generate karein.

Example:
**L1:** (ab + ba)*
**L2:** (a + b)*

Yaha par L1 aur L2 equivalent nahi hain kyunki L2 saari possible strings generate karta hai, lekin L1 sirf specific sequences generate karta hai.

4. Properties of Regular Languages

Regular languages kuch important properties ko follow karti hain:

  • **Closure Property:** Regular languages union, intersection, complement, aur concatenation par closed hoti hain.
  • **Decidability:** Regular languages ke liye sabhi algorithms (jaise DFA banane ke liye) decide kiye ja sakte hain.
  • **Finite Automata Representation:** Regular languages ko DFA se represent kiya ja sakta hai.

5. Pumping Lemma

Pumping lemma ek tarika hai jo yeh prove karta hai ki koi language regular nahi hai.

**Pumping Lemma:** Agar ek language regular hai, toh usme har ek string jo sufficient length ki hai, usme ek portion (substring) aisa hoga jise hum repeat kar sakein aur string ab bhi language me hogi.

Agar koi string pumping lemma ke according nahi hoti, toh wo language regular nahi hai.

6. Relationship Between DFA and Regular Expressions

Regular languages ko DFA (Deterministic Finite Automata) aur regular expressions dono se represent kiya ja sakta hai.

**DFA:** Ek finite state machine hota hai jo input strings ko accept ya reject karta hai based on transitions between states.

**Regular Expressions:** Ek notation hai jo strings ke pattern ko define karta hai.

DFA aur regular expressions dono regular languages ko represent karte hain, aur kisi DFA ka equivalent regular expression hota hai, aur vice versa.
